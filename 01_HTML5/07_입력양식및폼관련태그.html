text와 관련된 input 태그의 타입

type="text"

- 한 줄 짜리 텍스드를 입력할 수 있는 텍스트 상자(기본값)
- 생략하거나, 잘못쓰면 자동으로 type="text"가 된다.
* maxlength : 글자수 제한 거는것 (필수적으로 걸어야 함)
* 속성 속성값을 구분하는 "" 는 표기해주자 *

아이디:

type = "password"

-비밀번호를 입력 할 수 있는 텍스트 상자
** required: 아이디랑 비밀번호 입력 안 했을때, 페이지가 넘어지않게 해준다.
** not null == required

* 검색을 지울수 있는 x가 생김

* email에 email 속성 값만 입력 가능
* phone 문자키패드 열리는것, 핸드폰에서 열림 (인터넷으론 x)

라디오 버튼과 체크박스 관련 input 태그의 타입

라디어버튼(type="radio")

- 제시한 여러개의 값들 중에서 오로지 한 개만 선택해야 될 때
* : 글자 눌렀을때, id값 선택됨 (필수) * 그룹 지을때, name을 같은 이름으로 해야 다중 선택이 안됨 - (name 속성값이 똑같은 것들끼리 하나의 그룹으로 지어짐) * checked: 자동으로 체크되는 옵션 (강제성 부여, 취소x) * value: 넘어가는 값 지정(없을시, on으로 넘어감) - 제출시 현재 선택된(checked) 라디오 버튼의 value값이 넘어감
(해당 라디오 버튼 선택시 어떤 값을 넘길껀지 value값을 명시해줘야한다!)

성별:

체크박스(type="checkbox")

- 제시한 여러개의 값들 중에서 여러개 선택 가능할 경우
- 제출시 현재 선택된(checkbox) 체크박스의 value값이 넘어감 (넘기고자 하는 value값 명시해줘야 함) * 그룹으로 묶여 있어도 중복 체크 가능함 * hobby 배열에 reading과 movie가 넘어올거임(String 배열로) ( hobby[reading, movie] ) - 동일한 키 값으로 여러 value 값이 넘어온 경우에는 스트링 배열에 담긴다.

select 태그와 option 태그

- 사용자에게 직접 키보드로 값을 입력받는게 아니라,내가 제시한 여러개 옵션(목록) 들 중에서 선택 할 수 있겠끔
* 핸드폰번호 앞자리/지역/생일/.. (자주 쓰이는 기능) - 제출시 현재 선택된(select) option 의 값이 넘어감
- 각각의 option에 value값을 명시하지 않았을 경우 content 영역의 값이넘어감 - value 값을 명시했을 경우 value 값이 넘어감 - 기본값 선택안함을 하고 싶을때: 1. 목록 제일 위에, selected 작성
